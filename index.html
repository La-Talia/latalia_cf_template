When/How: Use for any simple 0-to-n loop. The go-to loop for iterating n times.
rep(i, n) {
    cout << i << "\n";   // prints 0 1 2 ... n-1
}

// Count elements matching a condition
int cnt = 0;
rep(i, n) if (a[i] > 0) cnt++;
repe(i, s, e)copy
for i in range(s, e) ▼
When/How: Use when you need to start from a non-zero value. Exactly like Python range(s, e).
// Sum elements from index 2 to n-1
ll total = 0;
repe(i, 2, n) total += a[i];

// Print odd numbers 1..19
repe(i, 1, 20) if (i % 2) cout << i << " ";
repd(i, n)copy
for i in range(n-1,-1,-1) ▼
When/How: Reverse iteration. Use when processing from the end — common in DP, suffix arrays, stack simulation.
// Print array in reverse
repd(i, n) cout << a[i] << " ";

// DP: fill from right to left (0-1 knapsack)
repd(j, W+1) if (j >= w[i])
    dp[j] = max(dp[j], dp[j-w[i]] + v[i]);

contains(a, x)copy
x in a — O(n) ▼
When/How: Linear search. Fine for small arrays but TLEs on large inputs. Use uset for repeated lookups.
// OK for small n (< 1000)
if (contains(a, 7)) cout << "found";

// For repeated lookups on large arrays, use uset:

⚠ ⚠ This is O(n). If you call it in a loop → O(n²). Use uset<T> instead.

x in sorted_a — O(log n) ▼
When/How: Binary search on a SORTED array. Use this instead of contains() after sorting. 10x faster for large arrays.

if (contains_sorted(a, 5)) cout << "yes";  // true
if (contains_sorted(a, 4)) cout << "yes";  // false
⚠ Requires the array to be sorted first. After sort(all(a)), always prefer this over contains().

dict
map<K,V> dcopy

When/How: Ordered map — keys always sorted. Use when you need min/max key, iteration in order, or ordered output.
map<int,int> d;
d[3] = 10;
d[1] = 20;
each2(k, v, d) cout << k << " ";  // 1 3  (sorted!)

// First/last key in O(log n)

umap<K,V> dcopy
d = {} — O(1) avg ✅ ▼
When/How: Unordered map — hash table. Use for pure key lookup/insert when order doesn't matter. 3-5x faster than map.
umap<string,int> score;
score["Alice"] = 95;
score["Bob"]   = 87;
get(score, "Alice", 0);  // 95
get(score, "Carol", 0);  // 0 (default)
⚠ ⚠ umap has worst-case O(n) if hash collisions occur. Adversarial inputs can hack it. Use map for safety in contests.

When/How: Safe lookup with default. Use instead of d[key] when the key might not exist — d[key] inserts 0 for missing keys.

Math — Modular Arithmetic
math
mpow(a, b, mod)copy
pow(a, b, mod) ▼
When/How: Modular exponentiation. Use for any a^b mod p in number theory, combinatorics, or crypto problems.
mpow(2, 10, MOD);     // 1024
mpow(2, 1000000000, MOD); // fast — O(log b)

// Fermat's little theorem: a^(p-1) ≡ 1 (mod p)
mpow(a, MOD-2, MOD);  // = modinv(a)
modinv(a, mod)copy
— modular inverse ▼
When/How: Modular inverse: finds x such that a*x ≡ 1 (mod p). Required for modular division. mod must be prime.
// Modular division: a/b mod p = a * modinv(b) mod p
ll ans = (a % MOD) * modinv(b, MOD) % MOD;

// Example: 10/3 mod 1e9+7
modinv(3, MOD);   // 333333336
(10 * modinv(3, MOD)) % MOD;  // works!
⚠ ⚠ mod must be a prime number. modinv(a, MOD) = mpow(a, MOD-2, MOD) by Fermat's little theorem.
gcd(a, b) / lcm(a, b)copy

When/How: gcd is iterative (no stack overflow). lcm = a/gcd*b. Use for fraction reduction, cycle detection.
gcd(12, 8);    // 4

// GCD of whole array:
ll g = a[0];
each(x, a) g = gcd(g, x);

When/How: Update variable if smaller/larger. Returns true if updated. Essential for clean DP transitions.
// DP: shortest path relaxation
if (chmin(dist[v], dist[u] + w)) {
    pq.push({dist[v], v});
}

// Track running max:
int ans = 0;
each(x, a) chmax(ans, x);  // ans = max of array

Prefix Sum

When/How: Build prefix sum array in O(n), then answer range sum queries in O(1). The most common CP trick.

When/How: Find SMALLEST x in [lo,hi] where cond(x) is true. The classic "binary search on answer" pattern.

    int groups = 1, cur = 0;
    each(x, a) {
        if (cur + x > mid) { groups++; cur = 0; }
        cur += x;
    }
    return groups <= k;
});
